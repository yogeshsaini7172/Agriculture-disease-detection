OVERVIEW
--------
This document describes every file that was created or modified
to implement the Satellite Precision Agriculture feature using
Sentinel-2 NDVI data from Sentinel Hub.

The feature allows farmers to:
  1. Enter their farm latitude, longitude, and radius (or use GPS).
  2. Tap "Analyze Crop Health" to send data to the backend.
  3. The backend fetches real Sentinel-2 NDVI satellite imagery
     (or simulates realistic data in Demo Mode).
  4. An ML model classifies crop health and returns:
       - Prediction label  (e.g. "Healthy Vegetation")
       - NDVI statistics   (mean, max, min, std, pixel count)
       - Health score      (0-100)
       - Confidence        (0-1)
       - Severity          (e.g. "Optimal", "Critical")
       - Recommendations   (irrigation, nutrients, field actions)
  5. The Android app displays the full dashboard with cards,
     a health score progress bar, and detailed recommendations.

FILE 1: backend/services/sentinel_service.py
  STATUS: FULLY REWRITTEN
----------------------------------------------------------------
WHAT CHANGED:
  - Added _is_demo_mode() function:
      Returns True when SENTINEL_HUB_CLIENT_ID is not configured.

  - Added _simulate_ndvi_stats(lat, lon) function:
      Generates realistic, deterministic NDVI statistics seeded
      by the farm coordinates. The same lat/lon always produces
      the same simulated NDVI values. This lets the full pipeline
      work without real Sentinel Hub credentials.
      Simulated NDVI range: 0.30 - 0.72 (typical Indian farmland).

  - Added get_ndvi_stats(lat, lon, radius_m) — NEW ENTRY POINT:
      Unified function used by main.py.
      Internally calls either:
        a) _simulate_ndvi_stats() if no credentials (Demo Mode)
        b) get_sentinel_token() + fetch_ndvi_image() + 
           extract_ndvi_stats() if credentials are present (Live)
      Returns: { mean_ndvi, max_ndvi, min_ndvi, std_ndvi,
                 pixel_count, bbox, demo_mode }

  - Updated fetch_ndvi_image() time range:
      Changed "to" date from "2025-12-31" to "2026-12-31"
      to include the current year's satellite imagery.

  - Fixed all Unicode characters in print statements:
      Replaced → with -> to avoid charmap encoding errors on
      Windows Python 3.14.

WHY:
  The original code called get_sentinel_token() directly, which
  raised RuntimeError and returned HTTP 503 when no Client ID
  was configured. The app completely failed with a 503 error.

FILE 2: backend/ml/crop_health_ml.py
  STATUS: NEW FILE (created for this feature)
----------------------------------------------------------------
WHAT CHANGED / ADDED:
  - NDVI_CLASSES dictionary:
      Maps class index to label:
        0 = "Critical Stress"
        1 = "Moderate Stress"
        2 = "Healthy Vegetation"
        3 = "Sparse / Bare Soil"
        4 = "Water Body / No Crop"

  - _ADVICE_DB dictionary:
      Stores expert advice for each class:
        - severity label (emoji + text)
        - irrigation_needed (True/False)
        - irrigation_recommendation (what to do)
        - nutrient_action (fertilizer advice)
        - field_action (scouting/inspection advice)
        - next_check_days (when to re-analyze)

  - build_feature_vector(ndvi_stats, extra_inputs):
      Builds a 5-feature (or 9-feature with weather) vector:
        [mean_ndvi, max_ndvi, min_ndvi, std_ndvi,
         pixel_count_normalized, (temp, humidity,
         rainfall, soil_ph if provided)]

  - _rule_based_prediction(mean_ndvi):
      Rule-based fallback classifier:
        NDVI < 0.0    -> "Water Body / No Crop"  (conf 0.92)
        NDVI < 0.10   -> "Critical Stress"        (conf 0.88)
        NDVI < 0.20   -> "Moderate Stress"        (conf 0.80)
        NDVI < 0.40   -> "Sparse / Bare Soil"     (conf 0.74)
        NDVI < 0.75   -> "Healthy Vegetation"     (conf 0.91)
        NDVI >= 0.75  -> "Healthy Vegetation"     (conf 0.96)

  - predict(ndvi_stats, extra_inputs):
      Main function called by main.py.
      Returns full result dict:
        { prediction, confidence, severity,
          ndvi_health_score (0-100), recommendation }

WHY:
  Needed an ML module to interpret raw NDVI numbers into
  actionable crop health insights.

FILE 3: backend/main.py
  STATUS: MODIFIED
----------------------------------------------------------------
WHAT CHANGED:
  - Added import for crop_health_ml at the top:
      from ml import crop_health_ml

  - Added import for sentinel_service at the top:
      from services import sentinel_service

  - Replaced the old 3-step satellite pipeline in
    analyze_crop() endpoint (/api/analyze-crop POST):

    OLD (broken) flow:
      Step 1: calculate_bounding_box()
      Step 2: get_sentinel_token()    <-- raised RuntimeError
                                           if no credentials
                                           -> returned 503
      Step 3: fetch_ndvi_image()
      Step 4: extract_ndvi_stats()
      Step 5: crop_health_ml.predict()

    NEW (fixed) flow:
      Step 1: sentinel_service.get_ndvi_stats(lat, lon, radius)
              [handles Demo Mode internally - no 503 anymore]
      Step 2: crop_health_ml.predict(ndvi_stats, extra_inputs)
      Step 3: save_report() in background thread
      Step 4: return JSON response

  - extra_inputs extraction simplified to a one-line dict
    comprehension that filters out None values.

WHY:
  The old code returned HTTP 503 "Satellite authentication
  failed" whenever Sentinel Hub credentials were missing.
  The new code uses the unified get_ndvi_stats() which
  automatically falls back to simulation.

FILE 4: backend/db/mongo_db.py
  STATUS: REWRITTEN (graceful failure)
----------------------------------------------------------------
WHAT CHANGED:
  - Wrapped MongoClient() in try/except:
      Before: Called MongoClient(MONGO_URI) at module level.
              If MONGO_URI was wrong/missing, Python crashed
              during import before Flask even started.
      After:  Checks if MONGO_URI is set and not a placeholder.
              If missing: prints a warning and sets all
              collections to None (demo mode).
              All CRUD functions (get_user_by_email, create_user,
              save_report, save_stress_analysis) safely return
              None or a fake result when DB is unavailable.

  - Added _PLACEHOLDER_URIS set to detect unconfigured URIs:
      {"", "mongodb+srv://<username>:<password>@<cluster>..."}

  - Added 5000ms serverSelectionTimeoutMS to MongoClient
    so it doesn't hang for 30s on bad connections.

WHY:
  Without this fix the backend crashed immediately on startup
  because .env had a placeholder MONGO_URI. The server never
  reached the satellite endpoint at all.

      OLD: "http://10.232.82.1:5000/api/"
      NEW: "http://127.0.0.1:5000/api/"

================================================================
  HOW TO SWITCH FROM DEMO MODE TO LIVE SATELLITE DATA
================================================================

1. Register at: https://apps.sentinel-hub.com/
2. Go to User Settings -> OAuth Clients
3. Create a new OAuth Client
4. Copy the Client ID and Client Secret
5. Open backend/.env and fill in:
     SENTINEL_HUB_CLIENT_ID=<your_client_id>
     SENTINEL_HUB_CLIENT_SECRET=<your_client_secret>
6. Restart the backend server.
7. The system will automatically use real Sentinel-2
   satellite imagery instead of simulated data.
